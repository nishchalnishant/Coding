# Books

Notes and summaries from algorithm and interview books. Use these as **reference and deeper reading** alongside the [Foundations](../foundations/README.md) and [Patterns](../patterns/README.md) in this repo.

| File | Book | Relevance to SDE-3 |
|------|------|--------------------|
| [The Algorithm Design Manual](the-algorithm-design-manual.md) | Steven S. Skiena | Algorithm design, correctness, heuristics, job scheduling. Strong for “why” and problem-solving mindset. |
| [Elements of Programming Interviews in Python](elements-of-programming-interviews-in-python.md) | Adnan Aziz et al. | Study guide, interview lifecycle, book organization (Ch 4–14: DSA). Use with our foundations for problem lists. |
| [Dynamic Programming for Coding Interviews](dynamic-programming-for-coding-interviews.md) | — | Bottom-up DP, recursion → DP. Complements [DP patterns](../patterns/dp-advanced.md) and [foundations DP](../foundatio../dynamic-programming/README.md). |
| [Data Structures and Algorithms Using Python](data-structures-and-algorithms-using-python.md) | — | ADTs, data structures. Good for formal definitions; pair with [foundations/data-structures](../foundatio../ds/README.md). |
| [CP3](cp3.md) | Competitive Programming 3 | Typing speed, problem-type recognition, complexity analysis, testing. Useful for speed and contest-style practice. |

**Tip:** For interview prep, start with [SDE3_DSA_ROADMAP.md](../SDE3_DSA_ROADMAP.md) and foundations; use book notes when you need extra depth or a different explanation.
