# Patterns

Problem-solving patterns for coding interviews (Google SDE-2 friendly). Use with the Google track at [../google-sde2/README.md](../google-sde2/README.md) or the deeper SDE-3 roadmap at [../SDE3_DSA_ROADMAP.md](../SDE3_DSA_ROADMAP.md).

- **[LeetCode Patterns](leetcode-patterns.md)** — Condensed guide to standard patterns (array, search, linked list, tree, graph, DP, backtracking).
- **[Two Pointers & Sliding Window](two-pointers-sliding-window.md)** — When to use opposite-end vs same-direction two pointers; variable vs fixed sliding window; pattern recognition.
- **[DP Advanced Patterns](dp-advanced.md)** — 16 DP patterns (Fibonacci, Knapsack, LIS, LCS, Interval, Tree, Bitmask, Digit, etc.).
- **[Coding Patterns](coding-patterns.md)** — Additional pattern reference.
- **[Common Patterns](common-patterns.md)** / **[Common Patterns (Theta)](common-patterns-thita.ai.md)** — Alternative pattern lists.

## How to use (SDE-2)

1. Learn the **pattern triggers** (what phrases hint at the pattern).
2. Memorize the **invariant** (what stays true while pointers/queue/stack move).
3. Practice 3–5 canonical problems per pattern until you can code in one pass.
